6.8 Search

The Nav menu offers several options for searching through the frontmost Collection.

The Advanced Search option will bring up a new window, in which you may enter the text you wish to search for, and then select from a number of common fields to indicate which of these fields should be included in the search. You will also have an option to perform a case-sensitive search, or to perform a case-insensitive one.

The default search options are to search (if present) the Title, AKA, Link, Tags, Author (aka Creator) and Body fields, and to ignore case (e.g., a search for ‘amazon’ will match ‘Amazon’).

You may also vary the scope of the search.

  • The default is to search all Notes in the Collection.
  • If your Collection has a Seq field, then you can also elect to search within the scope of the currently selected Note: in other words, the selected Note and all other Notes for which the selected Note is a parent.
  • You can also search forward, meaning within the currently selected Note and all following Notes in the Collection (based on the current sort sequence).

Hit the OK button to initiate the Search and select the first Note in the Collection matching the search criteria.

A simpler option is to use the Search command to place your cursor in the Search field at the top of the frontmost Collection window, then type the text you wish to search for, and press Enter to start the search.

When using this simpler option, prefixing the search text with a hash mark (‘#’) will cause Notenik to look only in the Tags field for the search string.

No matter how you initiated the Search, you can use the Search Next command to look for the next matching Note in the Collection following the last one found.

After using Search Next one or more times, you may use Search Previous to back up to a previously displayed search result.

Note that, after using the Advanced Search Option, the options selected will persist until changed, or until the end of the Notenik session, even if you later initiate a search using the simpler method of specifying a new text string in the Search field at the top of the window.

If you select one of the limited scope options (before or following), and then perform a search, and then modify the search term, the search for the new term will be performed with the previously selected scope, but using the currently selected Note. So, for example, if you had previously selected within, and then searched (moving the current note selection), and then changed the search term, you would no longer be searching the same set of candidate notes.

Each visible occurrence of the search text will be highlighted on the Display tab, for each matching Note.

There is also an option beneath the Nav menu to Tag Search Results.... This command can assign the user-specified tag to all Notes matching the search criteria. Of course, after assigning such a tag, the list of Notes containing that tag can be easily referenced on the Tags tab.
